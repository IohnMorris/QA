# Позитивные тест-кейсы: Форма Text Box (DemoQA)

**Тестовая среда:** https://demoqa.com/text-box
**Предусловие для всех тестов:** Открыта страница `https://demoqa.com/text-box`
**Цель:** Проверка того, что система корректно работает при вводе разрешенных данных в рамках установленных лимитов.

---

### TC-001 — Стандартное заполнение всех полей

**Шаги:**
1. Ввести `Ivan Ivanov` в поле **Full Name**
2. Ввести `ivan@mail.com` в поле **Email**
3. Ввести `Moscow, Tverskaya 1` в поле **Current Address**
4. Ввести `Moscow, Arbat 10` в поле **Permanent Address**
5. Нажать кнопку **Submit**

**Ожидаемый результат:**
Форма успешно отправляется. Под формой появляется блок с введенными данными, значения полностью соответствуют введенным.

---

### TC-002 — Граничные значения (минимально допустимый ввод)

**Шаги:**
1. Ввести `A` в поле **Full Name**
2. Ввести `a@b.co` в поле **Email**
3. Ввести `B` в поле **Current Address**
4. Ввести `C` в поле **Permanent Address**
5. Нажать кнопку **Submit**

**Ожидаемый результат:**
Форма успешно отправляется. Под формой отображаются все введенные одиночные символы и короткий email.

---

### TC-003 — Проверка безопасности (Санитизация XSS и HTML)

**Шаги:**
1. Ввести `12345` в поле **Full Name**
2. Ввести `test@test.com` в поле **Email**
3. Ввести `<b>Bold</b>` в поле **Current Address**
4. Ввести `<script>alert('XSS')</script>` в поле **Permanent Address**
5. Нажать кнопку **Submit**

**Ожидаемый результат:**
Форма успешно отправляется. Система корректно экранирует HTML-теги и скрипты: они отображаются под формой как обычный текст и не исполняются браузером.

---

### TC-004 — Отправка формы с пустыми (необязательными) полями

**Шаги:**
1. Оставить поле **Full Name** пустым
2. Оставить поле **Email** пустым
3. Ввести `Moscow, Tverskaya 1` в поле **Current Address**
4. Оставить поле **Permanent Address** пустым
5. Нажать кнопку **Submit**

**Ожидаемый результат:**
Форма успешно отправляется. Под формой отображается только заполненное поле: `Current Address: Moscow, Tverskaya 1`. Система корректно обрабатывает пустые значения без вызова ошибок.

---

### TC-005 — Граничные значения (максимально допустимый ввод)

**Шаги:**
1. Ввести строку из ровно **255** символов в поле **Full Name**
2. Ввести `ivan@mail.com` в поле **Email**
3. Ввести строку из ровно **1000** символов в поле **Current Address**
4. Ввести строку из ровно **1000** символов в поле **Permanent Address**
5. Нажать кнопку **Submit**

**Ожидаемый результат:**
Форма успешно отправляется. Под формой отображаются все введенные строки в полном объеме (без обрезания символов). База данных успешно принимает предельно допустимые значения.
