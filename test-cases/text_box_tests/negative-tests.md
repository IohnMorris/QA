# Негативные тест-кейсы: Форма Text Box (DemoQA)

**Тестовая среда:** https://demoqa.com/text-box
**Предусловие для всех тестов:** Открыта страница `https://demoqa.com/text-box`
**Цель:** Проверка того, как система обрабатывает невалидные данные, нарушение форматов и превышение установленных системных лимитов (см. Assumptions).

---

### TC-006 — Email без символа `@` (Неверный формат)

**Шаги:**
1. Ввести `Ivan Ivanov` в поле **Full Name**
2. Ввести `ivanmail.com` в поле **Email**
3. Нажать кнопку **Submit**

**Ожидаемый результат:**
Форма **не** отправляется. Блок с результатами под формой не появляется. Поле **Email** подсвечивается красным цветом (срабатывает фронтенд-валидация формата).

### TC-010 — Превышение лимита символов в поле Full Name (Boundary Limit)

**Шаги:**
1. Сгенерировать строку длиной **256** символов.
2. Ввести сгенерированную строку в поле **Full Name**
3. Ввести `ivan@mail.com` в поле **Email**
4. Нажать кнопку **Submit**

**Ожидаемый результат:**
Система не позволяет ввести более 255 символов (ввод блокируется на уровне UI). 
*Альтернативно:* Форма **не** отправляется, под полем отображается текст ошибки: "Превышено максимальное количество символов (max: 255)".

---

### TC-011 — Экстремальное превышение лимитов в текстовых полях (Stress/Limit Test)

**Шаги:**
1. Ввести `Ivan Ivanov` в поле **Full Name**
2. Ввести `ivan@mail.com` в поле **Email**
3. Вставить текст длиной **10 000** символов в поле **Current Address**
4. Вставить текст длиной **10 000** символов в поле **Permanent Address**
5. Нажать кнопку **Submit**

**Ожидаемый результат:**
Форма **не** отправляется. Система предотвращает отправку сверхтяжелого запроса на бэкенд. Поля адресов подсвечиваются ошибкой (превышен лимит в 1000 символов). Приложение не зависает и продолжает работать стабильно.
