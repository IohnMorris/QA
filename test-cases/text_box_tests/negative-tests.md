# 🛑 Негативные тест-кейсы: Форма Text Box

**Объект тестирования:** Элемент UI `Text Box`

**Тестовая среда:** [DemoQA Text Box](https://demoqa.com/text-box)

**Предусловие для всех тестов:** Открыта страница браузера по указанному URL.

**Цель:** Проверка обработки невалидных данных, нарушения форматов, превышения системных лимитов и базовой безопасности полей ввода.

| ID | Название (Title) | Шаги (Steps) | Ожидаемый результат (Expected Result) |
| :--- | :--- | :--- | :--- |
| **TC-TB-006** | **Email без символа `@` (Неверный формат)** | 1. Ввести `Ivan Ivanov` в поле **Full Name**.<br>2. Ввести `ivanmail.com` в поле **Email**.<br>3. Нажать кнопку **Submit**. | Форма не отправляется. Поле **Email** подсвечивается красным цветом (срабатывает фронтенд-валидация). Блок результатов не появляется. |
| **TC-TB-007** | **Email без доменной зоны** | 1. Ввести `Ivan Ivanov` в поле **Full Name**.<br>2. Ввести `ivan@` в поле **Email**.<br>3. Нажать кнопку **Submit**. | Форма не отправляется. Поле **Email** подсвечивается красным цветом. Блок результатов не появляется. |
| **TC-TB-008** | **Email с пробелом внутри** | 1. Ввести `Ivan Ivanov` в поле **Full Name**.<br>2. Ввести `ivan @mail.com` в поле **Email**.<br>3. Нажать кнопку **Submit**. | Форма не отправляется. Поле **Email** подсвечивается красным цветом. |
| **TC-TB-009** | **Email с двойным символом `@`** | 1. Ввести `Ivan Ivanov` в поле **Full Name**.<br>2. Ввести `ivan@@mail.com` в поле **Email**.<br>3. Нажать кнопку **Submit**. | Форма не отправляется. Поле **Email** подсвечивается красным цветом. |
| **TC-TB-010** | **Превышение лимита символов (Boundary Limit)** | 1. Сгенерировать строку длиной **256** символов.<br>2. Вставить её в поле **Full Name**.<br>3. Ввести `ivan@mail.com` в поле **Email**.<br>4. Нажать кнопку **Submit**. | Приложение не позволяет ввести более 255 символов (ввод 256-го символа блокируется на уровне UI).<br>*Альтернативно:* Форма не отправляется, появляется текст ошибки о превышении лимита. |
| **TC-TB-011** | **Экстремальное превышение (Stress Test)** | 1. Ввести `Ivan Ivanov` в поле **Full Name**.<br>2. Ввести `ivan@mail.com` в поле **Email**.<br>3. Вставить текст длиной **10 000** символов в поле **Current Address**.<br>4. Вставить текст длиной **10 000** символов в поле **Permanent Address**.<br>5. Нажать кнопку **Submit**. | Форма не отправляется, предотвращая отправку слишком тяжелого payload на сервер. Поля адресов подсвечиваются ошибкой. UI не зависает и продолжает работать стабильно. |
| **TC-TB-012** | **Отправка формы только с пробелами** | 1. Ввести 5 пробелов в поле **Full Name**.<br>2. Оставить остальные поля пустыми.<br>3. Нажать кнопку **Submit**. | Форма не воспринимает пробелы как валидные данные (пробелы отсекаются встроенной функцией `trim()`). Блок с результатами не появляется. |
| **TC-TB-013** | **Проверка на XSS (Cross-Site Scripting)** | 1. Ввести скрипт `<script>alert('bug')</script>` в поле **Full Name**.<br>2. Ввести `ivan@mail.com` в поле **Email**.<br>3. Нажать кнопку **Submit**. | JS-код **не** выполняется (alert-окно не появляется). Введенный текст безопасно экранируется и отображается в блоке результатов как обычный текст. |
| **TC-TB-014** | **Проверка на SQL-инъекцию** | 1. Ввести `Ivan Ivanov` в поле **Full Name**.<br>2. Ввести `ivan@mail.com` в поле **Email**.<br>3. Ввести `' OR 1=1 --` в поле **Current Address**.<br>4. Нажать кнопку **Submit**. | Данные обрабатываются системой как обычный текст и выводятся в блоке результатов. Ошибок базы данных (Syntax error) или раскрытия чужих данных не происходит. |
