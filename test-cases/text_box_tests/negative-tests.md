# 🛑 Негативные тест-кейсы: Форма Text Box

**Объект тестирования:** Элемент UI `Text Box`
**Тестовая среда:** [DemoQA Text Box](https://demoqa.com/text-box)
**Предусловие для всех тестов:** Открыта страница браузера по указанному URL.
**Цель:** Проверка обработки невалидных данных, нарушения форматов, превышения системных лимитов и базовой безопасности полей ввода.

| ID | Название (Title) | Шаги (Steps) | Ожидаемый результат (Expected Result) |
| :--- | :--- | :--- | :--- |
| **TC-TB-006** | **Email без символа `@` (Неверный формат)** | 1. Ввести `Ivan` в поле **Full Name**.<br>2. Ввести `ivanmail.com` в поле **Email**.<br>3. Нажать **Submit**. | Форма не отправляется. Поле **Email** подсвечивается красным цветом (срабатывает фронтенд-валидация). Блок результатов не появляется. |
| **TC-TB-007** | **Email без доменной зоны** | 1. Ввести `ivan@` в поле **Email**.<br>2. Нажать **Submit**. | Форма не отправляется. Поле **Email** подсвечивается красным цветом. |
| **TC-TB-008** | **Email с пробелом внутри** | 1. Ввести `ivan @mail.com` в поле **Email**.<br>2. Нажать **Submit**. | Форма не отправляется. Поле **Email** подсвечивается красным цветом. |
| **TC-TB-009** | **Email с двойным символом `@`** | 1. Ввести `ivan@@mail.com` в поле **Email**.<br>2. Нажать **Submit**. | Форма не отправляется. Поле **Email** подсвечивается красным цветом. |
| **TC-TB-010** | **Превышение лимита символов (Boundary Limit)** | 1. Сгенерировать строку длиной **256** символов.<br>2. Вставить её в поле **Full Name**.<br>3. Ввести `i@ya.ru` в **Email** и нажать **Submit**. | Приложение не позволяет ввести более 255 символов (ввод блокируется на уровне UI).<br>*Альтернативно:* Форма не отправляется, появляется текст ошибки о превышении лимита. |
| **TC-TB-011** | **Экстремальное превышение (Stress Test)** | 1. Вставить текст длиной **10 000** символов в **Current Address**.<br>2. Вставить **10 000** символов в **Permanent Address**.<br>3. Нажать **Submit**. | Форма не отправляется, предотвращая отправку тяжелого payload на сервер. Поля адресов подсвечиваются ошибкой. UI не зависает. |
| **TC-TB-012** | **Отправка формы только с пробелами** | 1. Ввести 5 пробелов в поле **Full Name**.<br>2. Нажать **Submit**. | Форма не воспринимает пробелы как валидные данные (пробелы отсекаются `trim()`). Блок с результатами не появляется или отображается пустым. |
| **TC-TB-013** | **Проверка на XSS (Cross-Site Scripting)** | 1. Ввести скрипт `<script>alert('bug')</script>` в поле **Full Name**.<br>2. Нажать **Submit**. | JS-код **не** выполняется (алерт не появляется). Введенный текст экранируется и безопасно отображается в блоке результатов как обычная строка. |
| **TC-TB-014** | **Проверка на SQL-инъекцию** | 1. Ввести `' OR 1=1 --` в поле **Current Address**.<br>2. Нажать **Submit**. | Данные обрабатываются как обычный текст. Ошибок базы данных (Syntax error) или непредвиденного поведения не происходит. |
