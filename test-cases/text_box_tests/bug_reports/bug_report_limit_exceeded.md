# [Bug]: Отсутствует ограничение на максимальное количество символов в полях ввода

**Labels:** `bug` `ui/ux` `security` 
**Priority:** High
**Severity:** Major
**Environment:** - OS: Windows 11 / macOS 14
- Browser: Chrome 122.0
- URL: https://demoqa.com/text-box

## Описание (Description)
В текстовых полях формы (Full Name, Current Address, Permanent Address) отсутствует валидация на максимальное количество вводимых символов. Система позволяет отправить форму с экстремально большим объемом текста (10 000+ символов), что может привести к переполнению базы данных (DB Overflow) или зависанию фронтенда на мобильных устройствах.

## Шаги для воспроизведения (Steps to Reproduce)
1. Открыть страницу https://demoqa.com/text-box
2. Сгенерировать текстовую строку длиной **10 000** символов (например, Lorem Ipsum).
3. Вставить строку в поле **Current Address**.
4. Нажать кнопку **Submit**.

## Ожидаемый результат (Expected Result)
Форма **не** отправляется. Система блокирует ввод свыше установленного лимита (например, 1000 символов для адреса) на уровне UI, либо выводит сообщение об ошибке: *"Превышено максимальное количество символов"*. Запрос на бэкенд не уходит.

## Фактический результат (Actual Result)
Форма успешно отправляется. Ошибок валидации не возникает. Под формой рендерится блок со всеми 10 000 введенными символами.
