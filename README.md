## 🚀 Практика: Тестирование Web-формы

**Объект тестирования:** [Text Box Form (DemoQA)](https://demoqa.com/text-box)

В этом проекте проведен анализ и тестирование простой веб-формы с применением техник тест-дизайна (анализ граничных значений, предугадывание ошибок). Особое внимание уделено не только UI-поведению, но и логике работы "под капотом" (ограничения баз данных, безопасность).

* 📁 **[Test Cases](./text-box-tests/)** — позитивные и негативные сценарии проверок с описанием ожидаемого результата по стандартам веб-разработки.
* 🐞 **[Bug Reports](./text-box-tests/bug_reports/)** — примеры оформления дефектов, включая симуляцию критической уязвимости к HTML-инъекциям (XSS).

## 📂 Структура репозитория

В данный момент репозиторий организован по модульному принципу (тесты и баги сгруппированы по тестируемым фичам):

* `test-cases/` — Корневая директория для тестовой документации.
  * `Text Box Test/` — Модуль тестирования формы Text Box (DemoQA).
    * `positive-tests.md` — Позитивные сценарии.
    * `negative-tests.md` — Негативные сценарии.
    * `bug_report/` — Оформленные дефекты (включая симуляцию уязвимостей), найденные в рамках этой формы.
    * `README.md` — Локальная документация модуля (допущения и стандарты проверок).
