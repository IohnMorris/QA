**Объект тестирования:** [DemoQA](https://demoqa.com/)

В этом проекте проведен анализ и тестирование веб-компонентов с применением техник тест-дизайна (анализ граничных значений, классы эквивалентности, предугадывание ошибок). Особое внимание уделено логике работы "под капотом" (ограничения, безопасность, API).

* 📁 **[Test Cases](test-cases/text_box_tests/)** — позитивные и негативные сценарии проверок с описанием ожидаемого результата по стандартам веб-разработки.

## 🗺️ План тестирования (DemoQA Roadmap)

В рамках данного репозитория я поэтапно покрываю тест-кейсами все основные UI-компоненты и страницы проекта DemoQA:

### 🧩 Elements (Базовые элементы)
- [x] **Text Box** — Позитивные/негативные проверки, лимиты, XSS-уязвимости.
- [ ] **Check Box** — Проверка состояний, работа с древовидными структурами (Tree View).
- [ ] **Radio Button** — Взаимоисключающий выбор, проверка неактивных (disabled) состояний.
- [ ] **Web Tables** — CRUD-операции (добавление, чтение, обновление, удаление), пагинация, поиск.
- [ ] **Buttons** — Обработка Double Click, Right Click и кнопок с динамическими ID.
- [ ] **Links** — Открытие в новых вкладках, перехват и проверка API-ответов (200, 201, 400, 404).
- [ ] **Broken Links - Images** — Валидация отображения картинок (Natural Width) и обработка битых ссылок (500).
- [ ] **Upload and Download** — Работа с файловой системой (загрузка/выгрузка файлов, проверка форматов).
- [ ] **Dynamic Properties** — Ожидание элементов, изменение состояний (цвет, видимость с задержкой).

### 📝 Forms (Формы)
- [ ] **Practice Form** — Сквозное (E2E) тестирование сложной формы регистрации (валидация, календари, радиокнопки, загрузка аватарок).

### 🪟 Alerts, Frame & Windows (Всплывающие окна и фреймы)
- [ ] **Browser Windows** — Работа с новыми вкладками и окнами (переключение контекста).
- [ ] **Alerts** — Нативные браузерные алерты (Alert, Confirm, Prompt), ввод текста в Prompt.
- [ ] **Frames** — Переключение фокуса внутрь iFrame и проверка контента.
- [ ] **Nested Frames** — Работа со вложенными фреймами (Parent -> Child).
- [ ] **Modal Dialogs** — Тестирование модальных окон (Small/Large), перекрытие интерфейса, закрытие по кнопке и фону.

### 🎛️ Widgets (Сложные виджеты)
- [ ] **Accordian** — Сворачиваемые/разворачиваемые панели, проверка анимаций.
- [ ] **Auto Complete** — Выпадающие списки с автодополнением (Single/Multiple colors).
- [ ] **Date Picker** — Выбор даты и времени, валидация форматов, переключение месяцев/лет.
- [ ] **Slider** — Интерактивный ползунок, проверка шага и граничных значений.
- [ ] **Progress Bar** — Динамическая полоса загрузки, старт/стоп, проверка 100%.
- [ ] **Tabs** — Переключение вкладок без перезагрузки страницы.
- [ ] **Tool Tips** — Всплывающие подсказки при наведении (Hover).
- [ ] **Menu** — Многоуровневые навигационные меню.
- [ ] **Select Menu** — Кастомные Dropdown-списки (Standard, Old Style, Multi-select).

### 🖱️ Interactions (Взаимодействия)
- [ ] **Sortable** — Сортировка списков и сеток (Drag and Drop перестановка).
- [ ] **Selectable** — Множественный выбор элементов кликом (List/Grid).
- [ ] **Resizable** — Изменение размеров блоков мышью (с ограничениями и без).
- [ ] **Droppable** — Перетаскивание объектов в заданную область (Simple, Accept, Prevent Propogation).
- [ ] **Dragabble** — Свободное перетаскивание элементов по странице (ограничения по осям X/Y).

### 📚 Book Store Application (Сквозное приложение)
- [ ] **Login** — Авторизация, сессии, токены.
- [ ] **Book Store** — Каталог товаров, фильтрация, добавление в коллекцию.
- [ ] **Profile** — Личный кабинет, удаление книг из коллекции.
- [ ] **Book Store API** — Тестирование бэкенда через Swagger/Postman (GET, POST, PUT, DELETE).
